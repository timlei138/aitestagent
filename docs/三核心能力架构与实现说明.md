# 三核心能力架构与实现说明

本项目当前按三个核心目标落地：

1. **语义全路径扫描**：先理解页面布局和入口，再扫描有意义路径。
2. **意图驱动测试**：根据用户意图自动操作设备并检查状态。
3. **回放**：按 YAML 用例或已记录路径重新执行并输出报告。

---

## 1. 语义全路径扫描

入口：

```bash
python main.py traverse --package com.android.settings --name Settings
```

核心实现：

- `core\smart_perceiver.py`
- `core\semantic_scanner.py`
- `core\baseline_store.py`
- `core\anomaly_detector.py`

扫描流程：

```text
启动 App
  ↓
SmartPerceiver 读取 UI 树和截图
  ↓
识别布局：左右分栏 / 单页 / 导航区 / 内容区
  ↓
给元素打标签：navigation_item / tab / settings_entry / switch / button
  ↓
SemanticScanner 按优先级点击主要路径入口
  ↓
记录页面截图、UI 树、元素、路径关系和异常
```

重点区别：

- 不再盲扫所有 clickable。
- 左侧导航栏、Tab、设置列表项优先。
- 右侧内容变化会被记录为从导航入口到详情区域的路径。
- 危险操作默认跳过，例如删除、提交、支付、重置。

## 2. 意图驱动测试

入口：

```bash
python main.py chat "检查 Settings 的 WiFi 开关打开和关闭"
```

核心实现：

- `core\intent_parser.py`
- `core\chat_runner.py`
- `core\agent.py`
- `core\tools.py`
- `core\smart_perceiver.py`

执行流程：

```text
用户自然语言
  ↓
IntentParser 解析 App、目标、动作、校验意图
  ↓
ChatRunner 构造测试任务
  ↓
Agent 使用工具操作设备
  ↓
每步读取页面语义和健康状态
  ↓
输出 PASS / FAIL 和报告
```

Wi-Fi 例子会被识别为：

```json
{
  "intent": "run",
  "test_type": "toggle_check",
  "target": "Wi-Fi",
  "app_package": "com.android.settings"
}
```

## 3. 回放

入口：

```bash
python main.py run_case --case test_cases\wifi_toggle.yaml
python main.py replay --case test_cases\wifi_toggle.yaml
```

核心实现：

- `core\replay_runner.py`
- `core\report_builder.py`
- `test_cases\wifi_toggle.yaml`

回放流程：

```text
加载 YAML
  ↓
启动 App
  ↓
按步骤点击、输入、等待、断言
  ↓
每步记录执行结果
  ↓
输出 JSON 报告
```

## 4. Web 控制台

入口：

```bash
python main.py server
```

访问：

```text
http://localhost:8080
```

能力：

- Chat 输入测试需求。
- 展示可编辑 Intent 确认卡片。
- 执行语义扫描、意图测试或回放。
- 设备截图预览。
- 页面语义和主要路径入口展示。
